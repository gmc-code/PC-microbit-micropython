Question 1
----------

.. multichoice::

    By default, what integer value does the sequence generated by the ``range(n)`` function start at?
    [x] 0 | Correct. It returns a sequence of numbers starting from 0 by default.
    [ ] 1 | Incorrect. In Python, ranges and indices start counting at 0 by default rather than 1.
    [ ] -1 | Incorrect. Negative starting integers must be explicitly defined in multi-argument ranges.
    [ ] It starts at a completely random integer. | Incorrect. The function generates a sequence spefically for the value of n.

Question 4
----------

.. multichoice::

    If you run the code below, what is the final number scrolled across the micro:bit LED display panel?

    .. code-block:: python

        from microbit import *

        while True:
            for n in range(3):
                display.scroll(n, delay=80)
            sleep(500)

    [ ] 3 | Incorrect. The range(3) statement halts execution before reaching 3.
    [x] 2 | Correct. The sequence contains 0, 1, and 2, making 2 the final value displayed in the block.
    [ ] 1 | Incorrect. The sequence continues past 1 to include the index 2.
    [ ] 0 | Incorrect. 0 is the starting number scrolled, not the final one.

    [ ] display.scroll() | Incorrect. This function animates data values on the screen but does not manage loops.
    [ ] sleep() | Incorrect. This function pauses execution timing instead of generating loop counts.
    [x] range() | Correct. To loop through a set of code a specified number of times, use the range() function.
    [ ] loop() | Incorrect. There is no built-in function named loop() for sequence generation in Python.

Question 7
----------

.. multichoice::

    Which of the following range code fragments correctly outputs the precise numeric list ``0, 3, 6, 9``?
    [x] .. code-block:: python

            for n in range(4):
                display.scroll(n * 3, delay=80)
        | Correct. range(4) generates 0, 1, 2, 3. Multiplying each by 3 gives 0, 3, 6, 9.
    [ ] .. code-block:: python

            for n in range(3):
                display.scroll(n * 3, delay=80)
        | Incorrect. This loop stops early, only outputting 0, 3, and 6.
    [ ] .. code-block:: python

            for n in range(4):
                display.scroll(n + 3, delay=80)
        | Incorrect. This adds 3 instead of scaling, outputting 3, 4, 5, 6.
    [ ] .. code-block:: python

            for n in range(9):
                display.scroll(n, delay=80)
        | Incorrect. This outputs all consecutive integers from 0 up to 8.

Question 8
----------

.. multichoice::

    Which code pattern produces a negative decreasing sequence displaying ``0, -1, -2, -3`` sequentially?
    [ ] .. code-block:: python

            for n in range(4):
                display.scroll(n, delay=80)
        | Incorrect. This creates a positive sequence containing 0, 1, 2, 3.
    [x] .. code-block:: python

            for n in range(4):
                display.scroll(n * -1, delay=80)
        | Correct. range(4) creates 0, 1, 2, 3. Multiplying by -1 converts them into 0, -1, -2, -3.
    [ ] .. code-block:: python

            for n in range(-4):
                display.scroll(n, delay=80)
        | Incorrect. Passing a single negative integer to range() returns an empty sequence.
    [ ] .. code-block:: python

            for n in range(3):
                display.scroll(n * -1, delay=80)
        | Incorrect. This loop terminates early, displaying only 0, -1, and -2.
